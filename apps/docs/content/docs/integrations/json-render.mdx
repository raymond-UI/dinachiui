## Overview

`@dinachi/json-render` is a first-party adapter for [json-render](https://json-render.dev/) (Vercel Labs' generative UI framework). It lets AI models generate JSON specs that render as fully accessible DinachiUI components.

The package provides:

- **Raw definitions** — Zod-validated schemas + component implementations, composable into any existing json-render project
- **Pre-built registry** — Ready-to-use `catalog` and `registry` for new projects (convenience)

## Installation

```bash
npm install @dinachi/json-render @json-render/core @json-render/react zod
```

## Quick Start

```tsx
import { Renderer, JSONUIProvider } from "@json-render/react";
import { registry } from "@dinachi/json-render";
import type { Spec } from "@json-render/core";

// Define a UI spec (typically generated by an AI model)
const spec: Spec = {
  root: "card",
  elements: {
    card: {
      type: "Card",
      props: { title: "Welcome", description: "Get started with DinachiUI" },
      children: ["greeting", "actionBtn"],
    },
    greeting: {
      type: "Badge",
      props: { text: "New", variant: "success" },
    },
    actionBtn: {
      type: "Button",
      props: { label: "Get Started" },
      on: { press: { action: "navigate", params: { url: "/docs" } } },
    },
  },
};

// Render
function App() {
  return (
    <JSONUIProvider registry={registry} initialState={{}}>
      <Renderer spec={spec} registry={registry} />
    </JSONUIProvider>
  );
}
```

## Composing with an Existing Project

If your project already uses json-render with its own catalog and registry, import the raw definitions and spread them in:

```ts
import { defineCatalog } from "@json-render/core";
import { schema, defineRegistry } from "@json-render/react";
import {
  dinachiComponentDefinitions,
  dinachiActionDefinitions,
  dinachiComponents,
  dinachiActionHandlers,
} from "@dinachi/json-render";

// Merge Dinachi definitions into your catalog
const catalog = defineCatalog(schema, {
  components: {
    ...yourComponentDefinitions,
    ...dinachiComponentDefinitions,
  },
  actions: {
    ...yourActionDefinitions,
    ...dinachiActionDefinitions,
  },
});

// Merge Dinachi implementations into your registry
const { registry } = defineRegistry(catalog, {
  components: {
    ...yourComponents,
    ...dinachiComponents,
  },
  actions: {
    ...yourActionHandlers,
    ...dinachiActionHandlers,
  },
});
```

## Catalog API

### Raw definitions

The `./catalog` subpath exports pure Zod schemas with no framework dependencies — safe for server-side usage:

```ts
import {
  dinachiComponentDefinitions,
  dinachiActionDefinitions,
} from "@dinachi/json-render/catalog";
```

### Pre-built catalog

For convenience, a pre-built catalog (via `defineCatalog()`) is available from the main entry:

```ts
import { catalog } from "@dinachi/json-render";
```

### System prompt generation

Use `catalog.prompt()` to auto-generate a system prompt from the catalog for AI models:

```ts
import { catalog } from "@dinachi/json-render";

const systemPrompt = catalog.prompt({
  customRules: [
    "Always wrap the root in a Box with padding='lg'",
    "Start with a clear heading (Text variant='h3')",
  ],
});
```

For API routes, construct the catalog from raw definitions to avoid pulling in React:

```ts
import { defineCatalog, buildUserPrompt } from "@json-render/core";
import { schema } from "@json-render/react/schema";
import {
  dinachiComponentDefinitions,
  dinachiActionDefinitions,
} from "@dinachi/json-render/catalog";

const catalog = defineCatalog(schema, {
  components: dinachiComponentDefinitions,
  actions: dinachiActionDefinitions,
});

const systemPrompt = catalog.prompt({ customRules: [...] });
```

## Registry API

### Pre-built registry

The package exports a ready-to-use `registry` created via `defineRegistry()`:

```ts
import { registry, handlers, executeAction } from "@dinachi/json-render";
```

### Custom registry

For advanced use cases, build a custom registry from the raw components:

```ts
import { defineRegistry } from "@json-render/react";
import { catalog, dinachiComponents, dinachiActionHandlers } from "@dinachi/json-render";

const { registry } = defineRegistry(catalog, {
  components: {
    ...dinachiComponents,
    // Override specific components
    Button: ({ props, emit }) => (
      <button className="custom-btn" onClick={() => emit?.("press")}>
        {props.label}
      </button>
    ),
  },
  actions: dinachiActionHandlers,
});
```

## Actions

Actions are triggered by event bindings in the spec's `on` field.

| Action | Params | Description |
|--------|--------|-------------|
| **navigate** | `url`, `target?` | Navigate to a URL. Set `target: "_blank"` for new tab. |
| **submit** | `formId?` | Submit a form element by ID. |
| **showToast** | `title`, `description?`, `variant?`, `timeout?` | Show a toast notification. |

Example event binding:

```json
{
  "type": "Button",
  "props": { "label": "Save" },
  "on": {
    "press": {
      "action": "showToast",
      "params": { "title": "Saved!", "variant": "success" }
    }
  }
}
```

## State Binding

Input components support two-way state binding via the `statePath` prop. State paths use JSON Pointer syntax (e.g., `/form/email`).

```json
{
  "root": "form",
  "state": {
    "form": { "email": "", "agreed": false }
  },
  "elements": {
    "form": {
      "type": "Card",
      "props": { "title": "Sign Up" },
      "children": ["emailInput", "agreeCheck"]
    },
    "emailInput": {
      "type": "Input",
      "props": {
        "label": "Email",
        "type": "email",
        "statePath": "/form/email"
      }
    },
    "agreeCheck": {
      "type": "Checkbox",
      "props": {
        "label": "I agree to terms",
        "statePath": "/form/agreed"
      }
    }
  }
}
```

## Validation

Form components support validation via `checks` and `validateOn` props:

```json
{
  "type": "Input",
  "props": {
    "label": "Email",
    "type": "email",
    "statePath": "/form/email",
    "checks": [
      { "type": "required", "message": "Email is required" },
      { "type": "email", "message": "Invalid email address" }
    ],
    "validateOn": "blur"
  }
}
```

Validation timing options: `"change"` (real-time), `"blur"` (on focus loss, default for Input/Textarea), `"submit"` (on form submission).

## Full App Example

```tsx
"use client";

import { Renderer, JSONUIProvider } from "@json-render/react";
import { registry, toastManager } from "@dinachi/json-render";
import type { Spec } from "@json-render/core";
import { Toast } from "@/components/ui/toast";

export function GenerativeUI({ spec }: { spec: Spec }) {
  return (
    <JSONUIProvider
      registry={registry}
      initialState={spec.state ?? {}}
    >
      <Renderer spec={spec} registry={registry} />
      <Toast toastManager={toastManager} />
    </JSONUIProvider>
  );
}
```
